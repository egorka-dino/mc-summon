# mc-commands — генераторы команд Minecraft 1.21.5+

Веб-инструмент для создания команд `/summon` и `/give` в Minecraft Java Edition (1.21.5 и новее). Полностью на русском языке.

## Возможности

### /summon

- **Все мобы игры** — 80+ существ, сгруппированных по категориям (враждебные, боссы, нейтральные, мирные, прочее)
- **Снаряжение** — все 6 слотов: шлем, нагрудник, штаны, ботинки, правая рука, левая рука
- **Зачарования** — список меняется под выбранный предмет, уровни от 1 до 255
- **Характеристики** — HP, скорость, броня, урон, сопротивление отбрасыванию, масштаб
- **Особенности** — ребёнок, неуязвимый, бесшумный, без ИИ, горящий/негорящий, светящийся, без гравитации, не исчезает
- **Эффекты зелий** — 32 эффекта с настройкой силы и длительности
- **Имя** — с выбором цвета, жирным и курсивным начертанием
- **Пассажиры** — моб верхом на мобе, любая глубина вложенности
- Команда обновляется автоматически в реальном времени
- Предупреждение, если команда длиннее 256 символов (нужен командный блок)

### /give

- **Предметы Minecraft 1.21.5+** — оружие, инструменты, броня, еда, зелья, блоки, материалы и особые предметы
- **Компоненты предметов** — имя, лор, зачарования, окраска кожаной брони, отделка брони, щиты, еда, тотем-поведение, фейерверки и зелья
- **Коллекция** — сохранение любимых предметов в `localStorage`
- **Случайный предмет** — генерация случайной полезной комбинации параметров
- **Длинные команды** — предупреждение после 256 символов и авто-замена `@s` на `@p` для командного блока

## Технологический слой

Сайт работает как Next.js App Router приложение на React и TypeScript:

- `/` — React-лендинг
- `/summon` — React-редактор команды `/summon`
- `/give` — React-редактор команды `/give`
- `/api/health` проверяет, что backend-слой Next.js отвечает
- `/api/db/health` проверяет подключение к Neon Postgres
- `/sign-in` и `/sign-up` отдают готовые страницы Clerk
- `/admin` показывает админку для пользователей с ролью `admin`: проверки и редактор общих шаблонов мобов
- `/api/summon/templates` отдаёт общие шаблоны для `/summon`
- `/api/admin/summon/templates` позволяет администратору создавать, изменять и удалять DB-правки шаблонов
- `/api/auth/google` перенаправляет на Clerk sign-in для старых ссылок
- `/api/auth/status` показывает готовность Clerk и текущего пользователя

Общие стили лежат в `style.css`, а данные и генерация команд разнесены по компонентам в `app/components/summon/` и `app/components/give/`.

### Локально

```bash
npm install
npm run dev
```

Для базы данных и авторизации скопировать `.env.example` в `.env.local` и заполнить значения:

```bash
cp .env.example .env.local
```

База данных подключена через Vercel Marketplace Neon. Локальные переменные можно обновить из Vercel:

```bash
npx vercel@latest env pull .env.local
```

### Clerk-логин

Авторизация сделана через Clerk. Для локальной разработки нужны переменные:

```bash
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=
CLERK_SECRET_KEY=
NEXT_PUBLIC_CLERK_SIGN_IN_URL=/sign-in
NEXT_PUBLIC_CLERK_SIGN_UP_URL=/sign-up
```

Google-вход включается в панели Clerk: **User & Authentication** → **Social connections** → **Google**. OAuth client ID/secret хранит Clerk, а не приложение.

Администратор задаётся в Clerk через metadata пользователя. В панели Clerk открыть нужного пользователя и добавить в **Public metadata** или **Private metadata**:

```json
{
  "role": "admin"
}
```

Также поддерживаются варианты `{ "admin": true }` и `{ "roles": ["admin"] }`. Сайт не доверяет `unsafeMetadata`, потому что эти данные может менять сам пользователь.

В Clerk нужно добавить домены приложения:

```text
http://localhost:3000
https://mc-commands.vercel.app
https://mc-commands.egorka.fun
```

На Vercel ключи можно добавить вручную или через Vercel Marketplace интеграцию Clerk.

## Как выложить на Vercel

1. Зайти на [vercel.com](https://vercel.com) и войти через GitHub
2. Нажать **Add New...** → **Project**
3. Импортировать репозиторий `egorka-dino/mc-commands`
4. Framework Preset выбрать **Next.js**
5. Build Command оставить `npm run build`
6. Output Directory оставить пустым
7. Нажать **Deploy**
8. После деплоя открыть **Settings** → **Domains** и добавить `mc-commands.egorka.fun`
9. В DNS домена направить `mc-commands.egorka.fun` на Vercel по подсказке в интерфейсе Vercel
10. После этого каждый `git push` в `main` будет автоматически обновлять сайт на Vercel

Текущий production URL: [https://mc-commands.vercel.app](https://mc-commands.vercel.app)

Для кастомного домена `mc-commands.egorka.fun` Vercel ожидает DNS-запись:

```text
A mc-commands.egorka.fun 76.76.21.21
```

## Как пользоваться /summon

1. Выбрать моба в самом верху
2. Настроить всё, что нужно (имя, характеристики, снаряжение, эффекты, пассажиры)
3. Команда появляется снизу в зелёном окошке
4. Нажать **Копировать**
5. В игре открыть чат (T или /) и вставить
6. Если команда длиннее 256 символов — положить в командный блок:
   - `/give @s command_block` в чате
   - Поставить блок, кликнуть правой кнопкой
   - Вставить команду, нажать **Готово**
   - Активировать рычагом, кнопкой или нажимной пластиной

### Шаблоны мобов

Встроенные шаблоны остаются в React-данных `/summon` как fallback. Если настроен `DATABASE_URL`, админка создаёт таблицу `summon_mob_templates` в Neon и сохраняет правки там. Запись с тем же `id`, что у встроенного шаблона, подменяет его для всех пользователей; удаление DB-записи возвращает встроенный вариант.
