# Админская библиотека предметов и мобов — Design Spec

## Цель

Первая preview-фаза добавляет в админку библиотеку готовых предметов и мобов, переводит текущие шаблоны мобов в библиотечную модель и подготавливает основу для будущих сценариев.

Публичные страницы `/give` и `/summon` остаются доступными по тем же URL и продолжают работать как генераторы одной команды.

## Объём первой фазы

Входит:

- переименование общих редакторов в предметно-доменные компоненты;
- переиспользование редактора предмета на `/give` и в админской библиотеке предметов;
- переиспользование редактора моба на `/summon` и в админской библиотеке мобов;
- новая админская страница `Библиотека предметов`;
- новая админская страница `Библиотека мобов`;
- новая таблица `library_items`;
- новая таблица `library_mobs`;
- миграция публичных шаблонов `/summon` на источник `library_mobs`;
- удаление пользовательского редактирования ID из админского интерфейса;
- обновление пользовательской и технической документации;
- preview deployment в Vercel после успешной сборки.

Не входит:

- сценарии;
- пакетное выполнение нескольких команд;
- AI-заполнение сценариев;
- Telegram-объявление;
- сохранение старых админских ссылок как обязательная совместимость.

## Редакторы

`GiveEditor` переименовывается в `ItemEditor`. Он остаётся в зоне `app/components/give/`, потому что формат команды и публичный URL всё ещё называются `/give`, но экспортируемый React-компонент описывает предмет, а не страницу.

`SummonEditor` переименовывается в `MobEditor`. Он остаётся в зоне `app/components/summon/`, потому что генерация команды всё ещё относится к `/summon`, но компонент описывает моба.

Оба редактора получают режимы, пригодные для публичной страницы и админки:

- публичный режим показывает текущие публичные возможности, включая preview, копирование, localStorage-избранное и существующее AI-заполнение;
- админский режим отключает публичное избранное и отдаёт snapshot наверх через `onSnapshotChange`;
- админский режим не копирует форму, а использует тот же редактор.

## Библиотека предметов

Админская страница: `/admin/library/items`.

В интерфейсе используются названия:

- `Библиотека`;
- `Библиотека предметов`;
- `Создать предмет`;
- `Редактировать`;
- `Дублировать`;
- `Удалить`;
- `Сохранить`.

Администратор видит список сохранённых предметов, выбирает запись или создаёт новую. Форма метаданных содержит название, категорию, описание, версию и флаг включения. ID не показывается как редактируемое поле.

Предмет хранит полный `GiveSnapshot`. Preview команды строится тем же `buildGiveCommand`, что и на публичном `/give`.

## Библиотека мобов

Админская страница: `/admin/library/mobs`.

Текущие `summon_mob_templates` больше не являются целевой моделью. В первой фазе создаётся `library_mobs`, и публичный `/summon` читает включённых мобов из этой таблицы как готовые шаблоны для выбора.

В интерфейсе используются названия:

- `Библиотека`;
- `Библиотека мобов`;
- `Создать моба`;
- `Редактировать`;
- `Дублировать`;
- `Удалить`;
- `Показывать на /summon`.

Моб хранит полный `SummonSnapshot`: `mobOrder` и `fields`. Preview команды строится тем же `buildSummonCommand`, что и на публичном `/summon`.

## Генерация ID

ID больше не задаётся пользователем.

При создании записи сервер генерирует стабильный ID. Базовая схема:

- префикс `item` или `mob`;
- slug из названия, если его можно безопасно получить;
- короткий уникальный суффикс.

При редактировании ID не меняется. При дублировании создаётся новая запись с новым ID.

## API и данные

Добавляются админские API:

- `GET /api/admin/library/items`;
- `POST /api/admin/library/items`;
- `DELETE /api/admin/library/items?id=...`;
- `GET /api/admin/library/mobs`;
- `POST /api/admin/library/mobs`;
- `DELETE /api/admin/library/mobs?id=...`.

Все маршруты доступны только администраторам через существующую Clerk-проверку.

Добавляется публичный API для `/summon`:

- `GET /api/summon/templates` возвращает включённые записи из `library_mobs`.

Если база данных недоступна, публичный `/summon` не должен падать: API возвращает пустой список, как сейчас.

## Навигация

Админская навигация получает ссылки:

- `Серверы Exaroton`;
- `Библиотека предметов`;
- `Библиотека мобов`.

Старый раздел `Шаблоны мобов` удаляется из навигации. Отдельная совместимость старого `/admin/templates` не обязательна.

## Документация

`docs/user-functionality.md` обновляется для админских сценариев:

- создание, редактирование, дублирование и удаление предметов в библиотеке;
- создание, редактирование, дублирование и удаление мобов в библиотеке;
- публичные `/give` и `/summon` остаются генераторами одной команды.

`docs/developer-documentation.md` обновляется для:

- новых компонентов `ItemEditor` и `MobEditor`;
- таблиц `library_items` и `library_mobs`;
- новых админских API;
- публичного чтения включённых мобов из `library_mobs`;
- удаления пользовательского ID из UI.

Блок `Что нового` на лендинге не обновляется, потому что первая фаза меняет только админку.

## Проверка

Перед preview:

- `npm run build` должен пройти;
- публичные `/give` и `/summon` должны собираться без TypeScript-ошибок;
- админские страницы библиотеки должны быть доступны только администраторам;
- Vercel preview создаётся командой `npx vercel deploy --yes`.

После preview пользователь проверяет окружение. Следующие фазы начинаются только после проверки и корректировок этой фазы.
