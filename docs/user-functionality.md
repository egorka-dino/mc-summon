# Пользовательский функционал mc-commands

Документация возможностей, которые видят обычные пользователи сайта `mc-commands` в публичном интерфейсе.

## Общая навигация

- Верхняя панель доступна на страницах сайта и ведёт в «Командный хаб», `/summon`, `/give` и «Серверная».
- Активный раздел подсвечивается в верхней панели.
- Статус входа и действия авторизации также находятся в верхней панели: вход, имя авторизованного пользователя и выход.
- Ссылка «Админка» показывается отдельно только пользователям с ролью `admin`.
- Во вкладке браузера используется favicon в тематике Minecraft-команд.

## Лендинг `/`

- Показывает название проекта и карточки навигации: `/summon`, `/give` и «Серверная».
- Содержит блок «Что нового» с релизами, сгруппированными по датам. Новые даты идут выше старых.
- Указывает версию сайта и целевую версию Minecraft Java Edition.

## Серверная `/servers`

- Показывает Minecraft-серверы сообщества, куда можно зайти поиграть.
- В карточке сервера видны название, адрес, статус, онлайн игроков и версия.
- Если доступных серверов нет, показывает пустое состояние без ошибки.

## Генератор `/summon`

Генерирует команду `/summon minecraft:<mob> <coords> ...` для создания мобов и цепочек пассажиров в Minecraft Java Edition 1.21.5+.

### Основные возможности

- Выбор координат спавна, по умолчанию `~ ~ ~`.
- Выбор моба из сгруппированного списка с поиском по русскому названию и ID.
- Общие шаблоны мобов, которые сразу заменяют текущего моба и пассажиров.
- AI-помощник «Собрать словами»: пользователь описывает моба обычным текстом, сайт заполняет подходящие поля редактора.
- Ответ AI-помощника обрабатывается в компактном формате, поэтому описание обычно применяется быстрее и без лишних служебных полей.
- Коллекция любимчиков в браузере: сохранение, загрузка, переименование, копирование команды и удаление настроек моба.
- Кнопка «Случайно» для генерации случайного моба с характеристиками, снаряжением, эффектами и пассажирами.
- Цепочка пассажиров: основной моб, пассажир на основном, следующий пассажир на предыдущем и так далее.
- Сворачиваемые карточки мобов и удаление пассажиров.
- Предпросмотр итогового моба: тип, имя, роль в цепочке, характеристики, флаги, снаряжение и эффекты.
- Автоматическое обновление готовой команды, копирование в буфер, сброс настроек и предупреждение при длине команды больше 256 символов.
- Для администраторов доступен блок «Выполнить на сервере»: сервер выбирается из списка онлайн-серверов Exaroton, игрок выбирается из списка онлайн-игроков выбранного сервера, а моб появляется рядом с этим игроком.
- Блок выполнения не показывается обычным пользователям и не содержит ручного ввода команд.
- Если сервер offline, список игроков недоступен или игроков нет онлайн, интерфейс показывает понятное сообщение и не даёт выполнить команду.
- Во время выполнения отображается loading-состояние, после ответа Exaroton показывается сообщение об успехе или ошибке.

### Настройки моба

- Имя, цвет имени, жирное и курсивное начертание, постоянная видимость имени.
- Характеристики: HP, скорость, броня, урон атаки, сопротивление отбрасыванию, масштаб.
- Флаги: ребёнок, неуязвимый, невидимый, бесшумный, без ИИ, не горит на солнце, постоянно горит, светится контуром, без гравитации, не исчезает, не задыхается.
- Флаг «Босс», который добавляет тег `boss` для серверного bossbar-мода.
- Специальные параметры фаерболлов: направление `Power` по X/Y/Z и мощность взрыва.
- Снаряжение: голова, нагрудник, ноги, ступни, основная рука и вторая рука.
- Количество предметов в слотах и шанс выпадения каждого слота.
- Зачарования с уровнями до `255`; обычный ванильный максимум показывается как подсказка, но не ограничивает команду.
- Окраска кожаной брони.
- Отделка брони.
- Узоры щитов и флагов: базовый цвет, до 16 слоёв, vanilla-узоры и цвета красителей.
- Эффекты зелий: выбор эффекта, сила `0-255`, длительность в тиках.

### Подсказки по команде

- Если команда длиннее 256 символов, её нужно использовать через командный блок.
- Пассажиры собираются вложенно: следующий моб сидит на предыдущем.
- Снаряжение мобов использует новый формат Minecraft 1.21.5+.

## Генератор `/give`

Генерирует команду `/give <target> minecraft:<item>[components] <count>` в формате Minecraft Java Edition 1.21.5+.

### Основные возможности

- Выбор цели: `@s`, `@p`, `@a`, `@r` или произвольный селектор/имя игрока.
- Выбор количества от 1 до 64 с жёстким ограничением значения.
- Выбор предмета из сгруппированного списка с поиском по русскому названию и ID.
- AI-помощник «Собрать словами»: пользователь описывает предмет обычным текстом, сайт заполняет подходящие поля редактора.
- Ответ AI-помощника обрабатывается в компактном формате, поэтому описание обычно применяется быстрее и без лишних служебных полей.
- Коллекция в браузере: сохранение, загрузка, переименование, копирование команды и удаление предмета.
- Кнопка «Случайно» для генерации предмета с подходящим количеством, именем, лором, чарами и особыми компонентами.
- Предпросмотр итогового предмета: имя, количество, цель, ID, лор, зачарования и включённые компоненты.
- Автоматическое обновление готовой команды, копирование в буфер и сброс настроек.
- Предупреждение при длине команды больше 256 символов.
- Если длинная команда использует цель `@s` без произвольного селектора, генератор автоматически меняет цель на `@p`, чтобы команда из командного блока выдала предмет ближайшему игроку, а не самому блоку.
- Для администраторов доступен блок «Выполнить на сервере»: сервер выбирается из списка онлайн-серверов Exaroton, игрок выбирается из списка онлайн-игроков выбранного сервера, а сгенерированный предмет выдаётся этому игроку.
- Блок выполнения не показывается обычным пользователям и не содержит ручного ввода команд.
- Если сервер offline, список игроков недоступен или игроков нет онлайн, интерфейс показывает понятное сообщение и не даёт выполнить команду.
- Во время выполнения отображается loading-состояние, после ответа Exaroton показывается сообщение об успехе или ошибке.

### Настройки предмета

- Кастомное имя, цвет имени, жирное и курсивное начертание.
- Лор до 5 строк, у каждой строки можно выбрать цвет.
- Зачарования, доступные для выбранного предмета, с уровнями до `255`.
- Окраска кожаной брони.
- Отделка брони: материал и узор.
- Узоры щитов и флагов: базовый цвет, до 16 слоёв, порядок слоёв, vanilla-узоры и цвета красителей.
- Параметры еды для edible-предметов: питательность, насыщение, возможность есть при полном голоде, до 5 эффектов при поедании с силой, длительностью и шансом.
- Поведение тотема бессмертия для любого предмета: до 3 эффектов после срабатывания и звук тотема.
- Фейерверки: длительность полёта, до 8 взрывов, форма, основные цвета, цвета затухания, след и мерцание.
- Именованные зелья: выбор эффекта, типа предмета и мощности.

### Зелья

- В списке предметов зелья представлены как именованные эффекты, например «Зелье силы».
- Тип зелья выбирается отдельно: обычное, бросаемое, длительное или зачарованная стрела.
- Мощность выбирается отдельно: обычное, продлённое или усиленное II, если вариант доступен для выбранного эффекта.

### Подсказки по команде

- Компоненты предметов собираются в квадратных скобках: `[comp=val,...]`.
- Для Minecraft 1.21.5+ не используется старый формат `levels` внутри enchantments.
- Если команда длиннее 256 символов, её нужно использовать через командный блок.

## Правило сопровождения

Если меняется поведение публичных страниц или пользовательских сценариев, этот документ нужно обновить в той же задаче.

Если изменение заметно обычным пользователям публичных генераторов, также нужно обновить блок «Что нового» на лендинге `/` согласно правилам из `AGENTS.md`.
